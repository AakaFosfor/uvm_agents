This document contains information regarding the classes, functions, members, definitions and constraints of this open source I2C UVM package.
The following describes the steps needed to integrate the I2C package into a working environment.
Version: 1.1
class i2c_base_sequence extends uvm_sequence #( i2c_sequence_item )
Inherited to all I2C master sequence classes.
class i2c_basic_sequence extends i2c_base_sequence
This sequence creates a number of constrainted I2C transactions to demonstrate the use of the I2C master agent.
class i2c_cfg extends uvm_object
I2C agent base configuration object.

class i2c_master_agent extends uvm_agent
Contains standard UVM agent objects including a configuration object, sequencer, driver and monitor.
class i2c_master_cfg extends i2c_cfg
I2C master agents configuration object.
class i2c_master_driver extends uvm_driver #( i2c_sequence_item )
Master driver begins driving an I2C transaction when an i2c_sequence_item is received from the agents sequencer and the I2C bus is not busy.
class i2c_monitor extends uvm_monitor
Monitors the I2C SDA and SCL lines and outputs via an analysis port an i2c_sequence_item.
The I2C package contains UVM master and slave agents capable of driving and monitoring a standard I2C bus.
class i2c_sequence_item extends uvm_sequence_item
The I2C sequence item encapsulates an entire transaction including multiple transmitted / received words for a single address request.
Sequence package for all I2C master sequences.
class i2c_slave_agent extends uvm_agent
Contains standard UVM agent objects for a slave agent including a configuration object, driver and monitor.
class i2c_slave_cfg extends i2c_cfg
I2C slave agents configuration object.
class i2c_slave_driver extends uvm_driver #( i2c_sequence_item )
Slave driver waits for a transaction to begin on the interface and responds in accordance with the i2c_slave_cfg configuration object members values.
